## 一、系統設計哲學

### 核心方法論：MK 以息養股

本系統基於「MK（郭俊宏）以息養股方法論」，核心理念為：

> **用配息收益買入更多單位數，以時間換取複利，讓資產自我滾動增長。**

與一般投資策略的差異：

| 一般投資思維 | MK 以息養股 |
|------------|-----------|
| 追求資本利得（低買高賣） | 追求持續配息現金流 |
| 頻繁換股（追熱點） | 長期持有（降低摩擦成本） |
| 市場情緒導向 | 景氣循環導向 |
| 個股集中 | 基金分散 |

### 核心/衛星雙軌架構

```
投資組合
├── 🛡️ 核心資產（目標 75%）
│   ├── 多元資產基金（配息為主）
│   ├── 投資等級債券基金
│   └── 高股息/平衡型基金
└── ⚡ 衛星資產（目標 25%）
    ├── 科技/AI 主題基金
    ├── 新興市場基金
    └── 產業主題基金（生技/能源等）
```

**核心判斷規則**（`_is_core_fund()`）：

| 類型 | 判斷依據 |
|-----|---------|
| 白名單 → 核心 | 安聯收益成長、聯博收益、富達多重資產 等 12 檔 |
| 關鍵字 → 核心 | 含「配息、債券、高股息、平衡、多元資產、income、bond」等 |
| 關鍵字 → 衛星 | 含「科技、AI、半導體、生技、印度、越南、成長、tech」等 |

### 2-2 FRED 指標抓取邏輯

所有 FRED 指標統一透過 `macro_engine._fred(sid, key, n=250)` 抓取：

```python
# 核心流程
fredapi.Fred(api_key=FRED_KEY).get_series(series_id, limit=n)
→ pd.Series（時間序列，按日期降序）
→ 計算 value（最新值）、prev（前值）、trend_slope（線性斜率）
→ 計算 z_score（相對過去 60 個月的標準化位置）
→ 計算 signal（📈上升 / 📉下降 / ➡️持平）
```

**Z-Score 計算方式**：
```
z_score = (當前值 - 過去60個月均值) / 過去60個月標準差
```

正值 = 高於歷史平均，負值 = 低於歷史平均。

## 三、總經位階判斷邏輯

### 3-1 核心評分模型（`calc_macro_phase()`）

12 個指標加權投票，正規化至 0~10 分：

| 指標 | 權重 | 加分條件 | 扣分條件 |
|-----|------|---------|---------|
| 殖利率利差 10Y-2Y | **2** | > 0.5% | < 0（倒掛） |
| 殖利率利差 10Y-3M | **2** | > 0.5% | < 0（最強衰退訊號） |
| ISM 製造業 PMI | **2** | > 50 且上升 | < 50 且下降 |
| HY 信用利差 | **2** | < 4% | > 6% |
| M2 貨幣供給 YoY | 1 | > 5% | < 0% |
| 市場廣度 RSP/SPY | 1 | RSP/SPY 上升 | RSP/SPY 下降 |
| Fed 資產負債表 | 1 | 擴表 | 縮表 |
| 美元指數 DXY | 1 | 弱勢 | 強勢突破 |
| VIX 恐慌指數 | 1 | < 20 | > 30 |
| CPI 通膨 | 0.5 | 下降趨勢 | 持續上升 > 4% |
| 聯準會利率 | 0.5 | 降息循環 | 升息 > 5% |
| 失業率 | 0.5 | < 4.5% 且穩定 | > 4.5% 且上升 |

**正規化公式**：
```
總分 = (已得加權分 + 最大可能分) / (2 × 最大可能分) × 10
```

### 3-2 景氣四象限判定

| 分數 | 景氣位階 | 顏色 | 建議配置 |
|-----|---------|------|---------|
| 8~10 | 🔴 高峰 | 紅 | 股35 / 債45 / 現金20 |
| 5~7 | 🟢 擴張 | 綠 | 股60 / 債30 / 現金10 |
| 3~4 | 🔵 復甦 | 藍 | 股40 / 債40 / 現金20 |
| 0~2 | 🟡 衰退 | 橙 | 股20 / 債50 / 現金30 |

### 3-3 二維位階確認（`get_market_phase()`）

以 PMI、殖利率利差、HY利差 三指標進行 Z-Score × 趨勢斜率雙軸判斷：

```
Z-Score 低（< -0.5）+ 斜率轉正（> +0.05）→ 復甦（最佳買點）
Z-Score 中位       + 斜率為正（> 0）     → 擴張（持有）
Z-Score 高（> +0.5）+ 斜率轉負（< -0.05）→ 減速（關鍵拐點！）
Z-Score 低          + 斜率為負（< 0）     → 衰退（防禦）
```

三個指標多數決，同時輸出信心度（投票比例）。

### 3-4 衰退機率計算

基於 10Y-3M 殖利率利差（聯準會官方衰退預測模型）：

```
logit = -1.5 × spread_10y3m - 0.8
衰退機率 = 1 / (1 + e^(-logit)) × 100%
```

殖利率深度倒掛時（spread < -0.5%），衰退機率 > 70%。

### 3-5 系統性風險掃描

每次載入總經資料同時抓取 RSS 財經新聞，透過關鍵字加權掃描評估系統性風險：

```
高危關鍵字（權重 5）：recession, crash, crisis, bankrupt, collapse
中危關鍵字（權重 3）：inflation, rate hike, default, stress, sell-off
低危關鍵字（權重 1）：slowdown, concern, caution, uncertainty

total_score = Σ(關鍵字權重 × 命中次數，單一關鍵字上限 3 次)

HIGH（≥10分）  → 🚨 建議立即降低風險暴露
MEDIUM（≥5分） → ⚠️ 密切追蹤，衛星設停利
LOW（<5分）    → ✅ 暫無系統性異常
```

## 四、單一基金分析邏輯

### 4-1 淨值與技術分析

從 MoneyDJ 抓取歷史淨值序列後，計算以下指標：

**布林通道（Bollinger Bands）**：
```
中軌 MA20 = 20日移動平均
上軌 = MA20 + 2σ  → 超買區（接近時考慮停利）
下軌 = MA20 - 2σ  → 超賣區（突破下軌 = 進場機會）
```

**MK 標準差買賣點**：
- 買1σ = 均值 - 1σ（優先加碼）
- 買2σ = 均值 - 2σ（強烈買進訊號）
- 停利 = 均值 + 1σ（設定停利點）

**NAV 位置評估**：
```
pos_label =
  NAV ≤ 買2σ → "深度超跌（2σ以下）" 🟢 強烈買進
  NAV ≤ 買1σ → "超跌（1σ以下）" 🟢 加碼
  NAV ≤ 均值 → "均值以下" 🟡 持有觀察
  NAV ≤ 停利 → "均值以上" 🟡 注意停利
  NAV > 停利 → "超買區（1σ以上）" 🔴 考慮減碼
```

### 4-2 六因子評分模型（`calc_fund_factor_score()`）

| 因子 | 權重 | 計算方式 | 說明 |
|-----|------|---------|------|
| Sharpe Ratio | **25** | `(-1~+1) → (0~100)` | 風險調整後報酬 |
| Max Drawdown | **20** | `0% → 100分; -30% → 0分` | 最大回撤（越小越好） |
| Alpha | **20** | `(-10%~+10%) → (0~100)` | 超額報酬（含息報酬-配息率） |
| Sortino Ratio | **15** | 同 Sharpe 計算 | 下行風險調整後報酬 |
| Calmar Ratio | **10** | `0~2x → 0~100` | 報酬 / 最大回撤比 |
| 費用率 | **10** | `0% → 100分; 3% → 0分` | 管理費越低越好 |

**評分等級**：A（≥75）/ B（≥55）/ C（≥40）/ D（<40）

### 4-3 配息安全分析（`dividend_safety()`）

核心問題：**基金是否在「吃本金」配息？**

```
覆蓋率 = 含息總報酬率(1Y) / 年化配息率

覆蓋率 < 0   → 🔴 嚴重吃本金（含息報酬為負）
覆蓋率 < 1.0 → 🔴 吃本金警示（配息 > 實際賺到的錢）
覆蓋率 < 1.2 → 🟡 邊緣（空間不足，密切觀察）
覆蓋率 ≥ 1.2 → 🟢 健康（配息有充分支撐）
```

### 4-4 HWM σ 絕對位階（`calc_hwm_sigma_levels()`）

以「歷史最高淨值（HWM）」為基準，計算 σ 絕對位階，用於評估當前淨值相對高點的位置：

```
HWM  = 過去 252 個交易日最高淨值
σ    = HWM × 年化日報酬率標準差 × √252

位階標準：
  HWM - 1σ → 中度回撤，加碼參考位
  HWM - 2σ → 深度回撤，分批承接訊號
  HWM - 3σ → 極端超跌，強烈左側布局訊號
```

與布林通道的差異：布林通道是相對移動平均的動態偏差；HWM σ 是相對歷史高點的絕對偏差。

## 五、投資組合管理策略

### 5-1 組合健康度監控

系統自動計算以下組合指標：

| 監控項目 | 計算方式 | 警戒條件 |
|---------|---------|---------|
| 核心/衛星比例 | 核心資產市值 / 總市值 | 偏離目標 > 10% 觸發警告 |
| 現金流估算 | Σ（單位數 × 淨值 × 配息率） | 配息率 < 4% 提醒 |
| 吃本金檢查 | 每檔基金覆蓋率計算 | 覆蓋率 < 1.0 亮紅燈 |
| 持倉相關性 | Pearson 矩陣（月報酬率） | 相關係數 > 0.85 = 影子基金警告 |

### 5-2 影子基金警告機制（`calc_correlation_matrix()`）

若兩檔基金月報酬率 Pearson 相關係數 > 0.85，視為持有同質性過高（影子基金）：

```
計算流程：
1. 各基金日淨值 → 重採樣為月末淨值
2. 計算月報酬率 pct_change()
3. 計算 Pearson 相關矩陣
4. 找出所有 |相關係數| > 0.85 的配對
5. 觸發警告：「XX 和 YY 高度相關，持有意義重疊」
```

### 5-3 最大化 Sharpe 比率最佳化（`optimize_portfolio()`）

當組合有歷史報酬序列時，使用 scipy 進行均值-變異數最佳化：

```
目標函數：最大化 Sharpe = (預期報酬 - 無風險利率) / 年化標準差
限制條件：
  - 各基金權重 0~60%（避免過度集中）
  - 所有權重加總 = 1
  - 無放空

求解：scipy.optimize.minimize（SLSQP 方法）
```

### 5-4 凱利公式資金管理（`calc_kelly()`）

計算基金歷史淨值序列的最佳資金投入比例：

```
資料：取最近 252 交易日日報酬率
計算：
  p  = 正報酬日比例（勝率）
  q  = 1 - p（敗率）
  b  = 平均獲利日 / 平均虧損日（賠率）
  f* = (b×p - q) / b（全凱利）

建議使用 Half-Kelly = f*/2（降低波動，實務更安全）
f* < 0 代表數學期望為負，建議不加碼
```

## 六、資產配置決策框架

### 6-1 景氣驅動的動態配置

```
景氣位階 × 個別基金訊號 → 操作建議

復甦期（分數 3~4）：
  核心資產 → 🟢 買進加碼（復甦期最高勝率買點）
  衛星資產 → 🟢 積極買進（成長資產進場黃金窗口）

擴張期（分數 5~7）：
  核心資產 → ⚪ 持有核心（繼續定期收息再投入）
  衛星資產 → 🟡 持有設停利（停利點 +10~15%）

高峰期（分數 8~10）：
  核心資產 → 🟡 持有減碼（轉向防禦性債券）
  衛星資產 → 🔴 賣出獲利（避免高基期風險）

衰退期（分數 0~2）：
  核心資產 → 🟢 逢低買進（配息資產在底部累積單位數）
  衛星資產 → ⏸️ 觀望等待（等 PMI 落底確認）
```

### 6-2 動態配比自動判斷

透過 PMI 和 VIX 即時計算建議股債比（`mk_fund_signal()`）：

```python
if PMI > 50 and VIX < 20:   → 股70% / 債30%  # 復甦/擴張積極
elif PMI > 50:               → 股60% / 債40%  # 擴張穩健
elif PMI < 50 and VIX > 25: → 股40% / 債60%  # 衰退保守
else:                        → 股50% / 債50%  # 觀望中性

額外條件：
  失業率 > 4.0% → 強制切換「衰退保守」(股40/債60)
  CPI 持續上升 > 3% → 「升息尾聲均衡」(股50/債50)
```

## 七、風險管理機制

### 7-1 複合風險溫度計（`PrecisionStrategyEngine`）

三指標加權計算宏觀風險評分：

```
Risk_Score = Z_VIX × 0.3 + Z_HY × 0.4 + Z_YC × 0.3

Z_VIX = (當前VIX - 歷史均值) / 歷史標準差
Z_HY  = (當前HY利差 - 歷史均值) / 歷史標準差
Z_YC  = (當前殖利率利差 - 歷史均值) / 歷史標準差
```

| Risk Score | 風險等級 | 建議現金部位 | 操作策略 |
|-----------|---------|-----------|---------|
| > 1.5 | 🚨 極高風險 | 50% | 現金/短債強制提高，嚴格停利 |
| > 0.8 | ⚠️ 風險偏高 | 30% | 縮減衛星至 20% 以內 |
| > 0.0 | 🔔 中性偏高 | 15% | 維持現有配置，設好停損 |
| > -0.5 | ✅ 中性偏低 | 10% | 正常配置，可適度加碼 |
| ≤ -0.5 | 🚀 風險極低 | 5% | 積極佈局，左側布局高成長衛星 |

### 7-2 三色燈號阻斷機制

資料完整率決定 AI 分析是否啟用：

```
資料完整率 = 成功載入指標數 / 總指標數 × 100%

🟢 ≥ 80%：AI 分析正常啟用
🟡 50~79%：AI 可執行，但結果參考性下降（顯示黃燈警告）
🔴 < 50%：AI 分析強制停用（紅燈阻斷）
```

**資料新鮮度退出加權（v19.351）**：短線風險雷達（`services/risk_radar.py`）的**個別燈號**
若最新資料點 **> 7 天**（對齊資料憲法 §2.4 日頻 🔴 門檻）視為過時 → 該燈轉 ⬜、**退出**
系統性風險紅/黃燈計數，**不以數週前的舊值誤判污染風險分數**（§1 不用過時值造假）。
首個落地點：**Put/Call 比率**（CBOE 源常過時，`_signal_put_call_ratio` 原只取 `iloc[-1]`

## 十、轉申購決策流程

### 10-1 換倉前必問三個問題

1. **換倉成本是多少？** → 使用 `calc_switch_cost()` 精確計算
2. **新基金需要多少報酬才能回本？** → 損益平衡門檻
3. **景氣位階支持這次換倉嗎？** → 參考 Tab1 景氣評分

### 10-2 轉換成本完整分解（`calc_switch_cost()`）

```
贖回金額 N 元
    ↓ × (1 - 贖回費%)       = 贖回後實得
    ↓ × (1 - 換匯價差%)     = 換匯後金額（相同幣別可省略）
    ↓ × (1 - 申購費%)       = 實際轉入基金 B 的金額

總成本率 = (N - 實際轉入) / N × 100%
損益平衡門檻 = 總成本率%（基金 B 未來報酬須超過此門檻才合算）
```

**典型案例**：
| 條件 | 計算結果 |
|-----|---------|
| 贖回 50 萬，贖回費 0%，申購費 3% | 損益平衡 3.0% |
| 贖回 50 萬，贖回費 0%，換匯價差 0.1%，申購費 3% | 損益平衡 3.1% |
| 贖回 50 萬，贖回費 1%，換匯 0.1%，申購費 3% | 損益平衡 4.1% |

### 10-3 換倉時機判斷原則

建議換倉的情境：
- 被換出基金的覆蓋率持續 < 1.0（長期吃本金）
- 被換入基金的 六因子評分 > 被換出基金 + 換倉成本臨界值
- 景氣位階支持換入標的的資產類型
- 被換出基金 NAV 在 HWM - 0.5σ 以上（相對高位）

不建議換倉的情境：
- 被換出基金 NAV 已跌破 HWM - 2σ（深度超跌，忍住）
- 換倉成本 > 3%（且被換入基金無明顯優勢）
- 景氣衰退期（衛星換衛星風險大）
