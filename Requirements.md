## 一、產品定位

**目標用戶**：個人投資者（非機構）
**核心價值**：整合台灣與美國市場資料，提供景氣判斷、個股深度分析、投資組合損益追蹤，以 AI 輔助決策。
**平台**：Streamlit Cloud（公開部署）+ GitHub（版本控制 + NAV 快取 CI/CD）

---

## 二、用戶故事與驗收標準

---

### US-01 總經儀表板（主頁）

**作為**投資者，**我希望**一眼看到當前景氣象限與資產配置建議，**以便**快速決定整體倉位方向。

#### 驗收標準
- [ ] 頁面載入後 30 秒內顯示景氣象限（擴張 / 過熱 / 衰退 / 復甦）
- [ ] 顯示 10 項指標燈號（綠/黃/紅），含 PMI、利差、CPI、VIX 等
- [ ] 依象限自動推薦股債比（例：衰退 → 股 30 / 債 70）
- [ ] FRED API 失敗時顯示離線快照並標示「⚠️ 離線模式，資料時間 N 小時前」
- [ ] 景氣分數 `infl_score` 計算公式透明（附說明文字）

---

### US-02 策略選股（pages/1）

**作為**投資者，**我希望**依景氣象限篩選對應策略的標的並看到即時報價，**以便**找到當前適合進場的標的。

#### 驗收標準
- [ ] 提供 6 大策略分頁（存股聖經 / VCP / 債券 ETF / 美股 ETF / AI半導體 / 台灣高息 ETF）
- [ ] 每檔標的顯示：代碼、名稱、最新價、漲跌幅、殖利率
- [ ] 漲跌幅遵守台灣慣例：🔴 漲 / 🟢 跌
- [ ] 殖利率 ≥ 7% 標紅、≥ 4% 標橘
- [ ] 資料快取 15 分鐘，提供強制刷新按鈕
- [ ] 單日漲跌幅超過 ±30% 時顯示「⚠️ 數據異常」而非直接顯示數值

---

### US-03 深度診斷（pages/2）

**作為**投資者，**我希望**輸入任意代碼後，取得技術面、籌碼面、財報、估值四大維度分析，**以便**決定是否進場。

#### 驗收標準

**技術面（K線）**
- [ ] 顯示 1 年日 K 線，含 MA20/60/120、布林通道、成交量、MACD
- [ ] AI 評斷：多頭 / 空頭 / 均線糾結 三態判斷，含行動建議
- [ ] 資料不足 60 日時顯示警告，不崩潰

**法人籌碼**
- [ ] 美股顯示機構持股前 15 大（yfinance）
- [ ] 所有標的顯示「量能代理指標」並標注「⚠️ 非真實三大法人資料」
- [ ] 台股缺乏真實法人資料時，顯示說明而非空白頁面

**財報獲利**
- [ ] 顯示近 8 季毛利率 / 營業利益率 / 淨利率趨勢折線圖
- [ ] 顯示單季 EPS 長條圖（紅正 / 綠負）
- [ ] ETF / 指數無財報時顯示「不適用」說明

**河流圖估值**
- [ ] EPS > 0：使用 PE 河流圖（5 年歷史 10/25/50/75/90 分位數）
- [ ] EPS ≤ 0：自動切換 PB 河流圖（0.5x / 1x / 1.5x / 2x / 3x）
- [ ] 兩者皆無：顯示「純 ETF/指數不適用估值河流圖」
- [ ] 白線為實際股價，所在色帶為估值象限

---

### US-04 庫存損益（pages/3）

**作為**投資者，**我希望**管理持倉並即時看到損益、MDD 與資產分佈，**以便**掌握整體組合健康度。

#### 驗收標準
- [ ] 可新增 / 移除持倉（代碼、資產類別、股數、平均成本）
- [ ] 持倉清單在頁面刷新後保留（session_state 持久化）
- [ ] KPI 卡片：總成本、總市值、未實現損益（含報酬率%）、持倉檔數
- [ ] 甜甜圈圖依資產類別顯示市值佔比
- [ ] 各標的報酬率長條圖（紅正 / 綠負）
- [ ] MDD 計算：過去 1 年最大回撤；超過 -20% 顯示額外警示
- [ ] 現價無法取得時顯示「—」，成本仍正確計入總計
- [ ] 強制刷新按鈕清除現價快取

---

### US-05 ETF 追蹤（主頁 Tab 6）

**作為**投資者，**我希望**集中追蹤台灣與美國 ETF 的即時報價與特殊訊號，**以便**判斷進出場時機。

#### 驗收標準
- [ ] 台灣預設 7 檔：0050 / 00878 / 00919 / 00929 / 006208 / 00646 / 00679B
- [ ] 美股預設 8 檔：SPY / QQQ / VTI / VOO / SCHD / IVV / GLD / TLT
- [ ] 顯示折溢價率：> +1% 🔴 警示 / < -1% 🟢 買入機會
- [ ] VCP 訊號：5 條件同時滿足時顯示「⚡ VCP 成立」+ 8% 停損價
- [ ] 可自訂新增 / 移除標的
- [ ] 資料快取 15 分鐘，提供強制刷新按鈕

---

### US-06 AI 投資建議（主頁 Tab 5）

**作為**投資者，**我希望**取得基於真實數值的 AI 總結報告，**以便**快速掌握行動清單。

#### 驗收標準
- [ ] 報告包含四章：景氣位階 / 資產配置建議 / 持倉警示 / 本週操作待辦
- [ ] AI 嚴禁捏造財報數字或未發生的新聞
- [ ] 每章不超過 300 字，條列式輸出
- [ ] API 失敗時顯示錯誤訊息，不顯示空白頁面
- [ ] 使用 Gemini 2.5 Flash 模型

### US-07 系統維運需求

**作為**開發者 / 部署者，**我希望**系統穩定運作並提供診斷工具，**以便**快速定位問題。

#### 驗收標準
- [ ] 側邊欄顯示 App / Engine / Fetcher 三組版本號
- [ ] 側邊欄顯示 FRED / Gemini / Proxy 連線狀態燈號
- [ ] 「🔍 測試 Proxy 連線」按鈕從 Streamlit Cloud 直接測試 NAS 可達性
- [ ] 「♻️ 強制同步」按鈕清除全部快取
- [ ] GitHub Actions 每日 00:30 台灣時間自動抓取 NAV 存入 `cache/nav/*.json`
- [ ] 所有報價元件下方顯示最後更新時間戳記

## 三、資料品質底線（不可妥協）

| 規則 | 說明 |
|------|------|
| 無虛假數值 | 所有數字必須來自外部 API，嚴禁硬寫假資料 |
| 空值不崩潰 | price = None / NaN / 0 時顯示「—」或「⚠️ 數據延遲」，不拋例外 |
| 極端值過濾 | 單日漲跌幅 > ±30%（VIX 除外）暫停訊號，標示「⚠️ 數據異常」|
| 快取標示 | 每個資料區塊下方顯示「最後更新：YYYY-MM-DD HH:MM TW」|
| 編碼正確 | MoneyDJ 全站 Big5 編碼，強制 `resp.encoding = "big5"` |

## 四、已知限制（Out of Scope）

| 項目 | 說明 |
|------|------|
| 台股三大法人 | yfinance 覆蓋率低；真實資料需 FinMind API（未來功能）|
| MoneyDJ IP 封鎖 | Streamlit Cloud 美國 IP 被擋；繞過方案為 NAS Proxy 或 GitHub Actions 快取 |
| 單一個股分析 | 本系統以基金 / ETF 為主；個股僅限策略選股清單中的預設標的 |
| 即時 Tick 數據 | 最小顆粒度為日線；不提供分鐘或 Tick 數據 |
